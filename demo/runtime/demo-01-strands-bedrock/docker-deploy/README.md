# Docker Deploy — AgentCore Starter Toolkit (CLI)

Alternative deployment path for Demo 1's agent using the **AgentCore Starter Toolkit CLI**.
Same agent code, deployed as a Docker container via `agentcore deploy`.

## Comparison: ZIP vs Docker (Starter Toolkit)

| | ZIP Deploy (`../deploy.py`) | Docker Deploy (this folder) |
|---|---|---|
| **Tool** | Python + uv + boto3 | AgentCore CLI (`@aws/agentcore`) |
| **Deploy command** | `python deploy.py` | `agentcore deploy -y` |
| **Build** | Local arm64 zip via uv | CLI handles Docker + ECR + CodeBuild |
| **API** | `codeConfiguration` (S3 zip) | `containerConfiguration` (ECR image) |
| **Cleanup** | `python cleanup.py` | `aws cloudformation delete-stack` |
| **Local deps** | Python + uv | Node.js + npm + AgentCore CLI |
| **Build time** | ~30s | ~3-5 min (includes CDK + Docker build) |
| **Best for** | Quick prototyping, classroom | Production, CI/CD pipelines |

## Prerequisites

```bash
# 1. Node.js 20+ (required for the CLI)
node --version   # must be 20+

# 2. Install the AgentCore CLI
npm install -g @aws/agentcore

# 3. Verify
agentcore --version

# 4. AWS CDK (used by the CLI for deployment)
npm install -g aws-cdk

# 5. Python 3.10+ and uv (for generating lock file)
python3 --version
uv --version

# 6. AWS credentials configured
aws sts get-caller-identity
```

**Required AWS permissions:** See [Use the AgentCore CLI](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/runtime-permissions.html#runtime-permissions-cli).

**Model access:** Enable Anthropic Claude Haiku 4.5 in the [Amazon Bedrock console](https://console.aws.amazon.com/bedrock/home#/modelaccess) for `ap-southeast-1`.

## First-Time Setup

Before the first deploy, generate the `uv.lock` file (required by the Dockerfile):

```bash
cd app/StrandsDemo
uv lock
cd ../..
```

If the CDK project is missing (`agentcore/cdk/` does not exist), regenerate it:

```bash
# Copy the CDK template from the CLI package
cp -r $(npm root -g)/@aws/agentcore/dist/assets/cdk agentcore/cdk
cd agentcore/cdk && npm install && cd ../..
```

## Usage

### Deploy

```bash
cd docker-deploy
bash deploy.sh
```

The wrapper script handles all setup (lock file, CDK, config restoration) then calls `agentcore deploy -y`.

### Invoke (via CLI)

```bash
# Single prompt
agentcore invoke "What is the weather in Seattle?"

# Multi-turn session
SESSION_ID=$(uuidgen)
agentcore invoke --session-id "$SESSION_ID" "My name is Alice"
agentcore invoke --session-id "$SESSION_ID" "What is my name?"

# Math calculation
agentcore invoke "What is 123 * 456 + 789?"
```

### Cleanup (safe for repeated cycles)

```bash
bash cleanup.sh
```

This deletes the CloudFormation stack and restores `agentcore.json` so the next deploy works cleanly.

### Repeatable Deploy/Cleanup Cycle

```bash
bash deploy.sh          # Deploy
agentcore invoke "Hi"   # Test
bash cleanup.sh         # Tear down
bash deploy.sh          # Deploy again — works every time
```

### Invoke (via parent folder's scripts — same API)

```bash
# Create a runtime_config.json with the ARN from deploy output, then:
cd ..
python invoke.py
python invoke_agent.py
```

### Local Development

```bash
agentcore dev
# Starts agent locally — test before deploying
```

### Check Status

```bash
agentcore status
```

## Troubleshooting

### `CDK project not found`

The `agentcore/cdk/` directory is generated by the CLI on first `agentcore create`. If it's missing (e.g., after cloning from git), regenerate it:

```bash
cp -r $(npm root -g)/@aws/agentcore/dist/assets/cdk agentcore/cdk
cd agentcore/cdk && npm install && cd ../..
```

### `uv.lock not found` (Docker build fails)

The Dockerfile expects a `uv.lock` file. Generate it:

```bash
cd app/StrandsDemo
uv lock
```

### `No resources defined in project` (empty runtimes)

The CLI mutates `agentcore.json` during remove operations, emptying the `runtimes` array. Use the wrapper scripts (`deploy.sh` / `cleanup.sh`) which auto-restore from git. Or manually restore:

```bash
git checkout -- agentcore/agentcore.json
```

```
    └── agentcore/state/        ← Deployment state
```

## Key Talking Points

1. **`agentcore deploy` = one command** — handles ECR, Docker build, push, CDK, runtime creation
2. **`agentcore dev` = local testing** — same HTTP contract as production
3. **`agentcore invoke` = test the deployed agent** — no separate client script needed
4. **Same agent code** — `main.py` is the same as `../agent.py`
5. **Same invoke API** — `invoke.py` from the parent folder works regardless of deploy method
6. **Production pattern** — CDK stack for infrastructure-as-code, container for reproducibility
